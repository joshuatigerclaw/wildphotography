# WildPhotography Internal Linking Rules

These rules apply to all content generated by `wild_article_generator`. Every article must satisfy these requirements before passing the quality gate.

---

## Minimum Link Requirements

| Link Type | Minimum | Maximum |
|---|---|---|
| Photo pages | 2 | 4 |
| Gallery pages | 2 | 3 |
| Species pages | 1 | 3 |
| Location or region pages | 1 | 3 |
| **Total minimum** | **5** | — |

---

## Link Validity Rules

- Only link to slugs confirmed present in the `link_targets` dataset
- Never link to a slug that does not exist in `link_targets`
- Never link to the same slug twice in one article
- Never fabricate slugs or guess at page paths
- If a logical link target does not exist in `link_targets`, omit the link rather than guessing

---

## Anchor Text Rules

- Always use descriptive anchor text that describes the destination page
- Never use: "click here", "read more", "this page", "here", or "learn more" as anchor text alone
- Preferred anchor text patterns:
  - Species: `Scarlet Macaw in Costa Rica`
  - Gallery: `Costa Rica Bird Photography Gallery`
  - Location: `Manuel Antonio Travel Guide`
  - Region: `Guanacaste Wildlife and Beaches`
  - Photo: `Wildlife Photography from Tortuguero`

---

## Placement Rules

- Place at least 3 internal links within article body paragraphs (not just in the Related Links section)
- Link to the strongest parent topic page and at least two related child pages
- Include a "Related Guides" section at the end of every article with 5+ links
- Do not cluster all links in one section — distribute naturally throughout the article

---

## Linking Priority

When selecting which pages to link to, prioritise in this order:

1. Directly related gallery page for the article's main subject
2. Species pages for any species named in the article
3. Location pages for any location named in the article
4. Region page for the broader region covered
5. Photography guide pages with complementary angles
6. Other published articles with non-overlapping intent

---

## URL Path Conventions

Use these URL path prefixes when constructing internal link hrefs:

| Page Type | Path Prefix |
|---|---|
| Gallery | `/galleries/{{ slug }}` |
| Species | `/species/{{ slug }}` |
| Location | `/locations/{{ slug }}` |
| Region | `/regions/{{ slug }}` |
| Article | `/articles/{{ slug }}` |
| Photo | `/photos/{{ slug }}` |

---

## Internal Links Output Format

Return internal links as a JSON array in the `internal_links` field:

```json
[
  {
    "slug": "birds-of-costa-rica",
    "page_type": "gallery",
    "anchor_text": "Costa Rica Bird Photography Gallery",
    "href": "/galleries/birds-of-costa-rica",
    "placement": "paragraph"
  },
  {
    "slug": "scarlet-macaw",
    "page_type": "species",
    "anchor_text": "Scarlet Macaw in Costa Rica",
    "href": "/species/scarlet-macaw",
    "placement": "related_links_section"
  }
]
```

`placement` values: `paragraph`, `related_links_section`, `caption`, `intro`

---

## Deduplication Across Articles

- Do not use identical anchor text for the same slug across multiple articles
- Vary the phrasing slightly to avoid repetitive patterns at scale
- Each article should have a distinct internal link profile even when targeting adjacent topics
